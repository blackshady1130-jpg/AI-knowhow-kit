# PPT One-Pager Writer

## Use When

- 用户要“一页PPT文案”“汇报页”“领导汇报摘要”。
- 用户明确要求精炼、结构化、可直接贴入PPT。

## Input Contract

必须先拿到这四项（缺失就追问）：

1. 汇报目标（这页想让谁做什么决策）
2. 主题对象（项目/业务/合作）
3. 证据要点（数据、里程碑、进展）
4. 风险或阻塞（没有可写“暂无关键阻塞”）

## Output Contract

按固定结构输出：

1. 标题（1句，结论式）
2. 三条支撑（每条 1-2 句）
3. 风险与前提（1-2 条）
4. 下一步（按时间或优先级）

## Quality Bar

- 不写空泛口号。
- 每条支撑必须有对象或结果。
- 避免“夸赞式”语气。
