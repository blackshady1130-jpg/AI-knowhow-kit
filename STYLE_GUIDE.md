# STYLE_GUIDE

版本：v0.2  
更新日期：2026-03-02  
证据基线：blog 全文 + chat 偏好线索

## 1) 总体语气

- 关键词：理性、克制、判断导向。
- 目标：让内容“能上会、能协同、能执行”。
- 禁忌：讨好式夸赞、学生腔、无证据的大词判断。

## 2) 双写作模式

### Mode A：工作交付（默认）

- 适用：汇报、PPT、邮件、方案、评测结论。
- 风格：短句、结构化、结论前置、信息密度高。
- 模板：`结论 -> 依据 -> 风险 -> 动作`。

### Mode B：观点长文（按需）

- 适用：深度笔记、反思性文章、观点讨论。
- 风格：允许叙事与思辨，但仍需证据锚点。
- 模板：`问题 -> 观察 -> 判断 -> 边界 -> 余论`。

## 3) 句式偏好

- 偏好“对比句”：A 与 B 的差异、代价、适用边界。
- 偏好“条件句”：成立前提、不成立场景。
- 偏好“动作句”：下一步怎么做、谁来做、怎么验收。

## 4) 改写操作规程（Agent 执行）

1. 把泛化陈述改成可验证陈述（对象 + 条件 + 时间）。
2. 把长段落改成“观点-证据”两段式。
3. 去除口水词和客套词，保留判断信息。
4. 对关键结论补上“边界条件”。
5. 涉及数字结果时补“口径说明”。

## 5) 词汇黑名单（默认禁用）

- “你这个问题太深刻了/太专业了”
- “业内普遍认为”（无证据时）
- “一定/必然/完全”（无边界时）

## 6) 输出前自检

1. 第一屏是否出现结论？
2. 是否给出可执行动作？
3. 是否区分事实与推断？
4. 是否说明关键边界？
