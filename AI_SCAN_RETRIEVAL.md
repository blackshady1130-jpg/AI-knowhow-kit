# AI_SCAN_RETRIEVAL

版本：v0.1  
更新日期：2026-03-13  
适用范围：Cursor / Claude Code / Codex / 通用 Agent

## 1) 触发条件（何时启用本协议）

当用户问题属于以下任一场景时，先启用本协议再作答：

- AI 行业公司、产品、战略、生态变化（如 OpenAI、Anthropic、Google、SaaS）
- 模型能力、训练范式、上下文工程、Agent 工程实践
- 评测/benchmark/方法论对比（含 in-domain / out-of-domain）
- AI 商业化、ROI、组织与交付模式判断

若问题不属于 AI 行业扫描主题，按常规知识与工具流程处理。

## 2) 数据资产与职责

- `notes/AI行业扫描_keywords.jsonl`：唯一主索引，用于候选召回。
- `notes/AI行业扫描_md/*.md`：原文证据，用于引用与核对。
- `notes/AI行业扫描.xlsx`：源数据底表（若存在），不作为默认检索入口。

## 3) 检索顺序（必须遵守）

1. **候选召回**：先在 `keywords.jsonl` 按关键词、标题、`why` 召回候选条目。  
2. **价值筛选**：用 `why` 与 `type` 判断相关性与可用性，保留最相关 3-8 条。  
3. **证据抽取**：仅在需要给出细节/原句时，打开对应 `md` 做定向摘取。  
4. **结构化回答**：输出 `结论 -> 证据 -> 边界 -> 下一步`，并附 `id/title/url`。

## 4) 回答约束

### 必须做到

- 明确区分：已确认事实 / 推断判断 / 待验证点。
- 引用证据时给出可追溯条目（`id/title/url`）。
- 对涉及趋势或策略判断的问题，写出适用边界和失效条件。

### 明确禁止

- 先把 `AI行业扫描_md` 全量通读后再回答。
- 无证据时给绝对化结论（如“必然”“一定”）。
- 只列观点，不给行动建议或下一步验证路径。

## 5) 最小执行模板

可直接复用以下输出骨架：

1. 结论：一句话判断。  
2. 证据：列 2-4 条高相关条目（`id/title/url` + 关键依据）。  
3. 边界：样本覆盖、时间窗口、潜在偏差。  
4. 下一步：建议补充检索方向或验证动作。  

## 6) 快速自检

- 是否先检索 `keywords.jsonl`，而非先读全部 `md`？
- 是否解释了“为什么选这些证据”？
- 是否给出边界与下一步，而不只给观点？

## 7) 与 bookmarks 的双库协同

当任务同时需要"判断质量 + 资料覆盖"时，按以下流程执行：

### 任务分类

1. 判断优先（strategy / trade-off / recommendation）：先 `AI_SCAN`，后 `BOOKMARKS` 补证
2. 资料优先（collect / compare / examples）：先 `BOOKMARKS`，再 `AI_SCAN` 做判断锚定
3. 混合型：两库并行召回，再统一筛选

### 执行步骤

1. 从两个索引（`notes/AI行业扫描_keywords.jsonl` + `bookmarks/bookmarks_keywords.jsonl`）并行召回候选，每库 5-10 条。
2. 按相关性、`quality_tier`、`why` 字段、时效性做筛选，收敛到 3-8 条。
3. 仅读取最终候选对应 md 文件，不做全量阅读。
4. 统一输出 `结论 -> 证据 -> 边界 -> 下一步`，并附 `id/title/url`。

### 默认策略

- 无额外要求时：`AI_SCAN` 权重高于 `BOOKMARKS`
- 发现冲突证据时：并列展示，不强行统一
- 若任一库无有效结果：保留另一库结果并标注"覆盖不足"
