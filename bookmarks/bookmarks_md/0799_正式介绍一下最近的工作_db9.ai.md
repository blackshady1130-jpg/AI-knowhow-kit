# 正式介绍一下最近的工作: db9.ai

dongxu dongxu | 我世界的源代码

db9 上了 Hacker News 首页。经过 3 个月闭关，燃烧几百亿 token（Opus 4.6 + GPT 5.2），得到了 db9.ai——一个给 AI Agent 使用的新型存储平台。

## 为什么今天还要做一个新 DB？

PG 不够香。对于 Agent 来说，目前两个主要的心智模型——SQL（PostgreSQL 的 SQL）和文件系统（以及 unix sh tools：ls/cp/grep/glob...）——一直是分裂的。

db9 将这两者融合：fs 和 sql 作为主要接口，但不是独立的，而是融合在一起。

```
$ db9 fs sh
fs9:/> echo hello > world
$ db9 db sql
db9:demo-site=> select * from extensions.fs9('/world');
```

## 设计理念

- **文件作为中心和入口**：数据只要进入，系统便可以理解一切——写入一切，然后查询一切（无代码，无 SQL）
- **打通非结构化→结构化数据的桥梁**
- **对 Agent 最简单的入口**：`cp -r ./docs:/docs`，然后剩下的事情交给 db9 和 Agent
- **SQL 也可以作为入口**，这是传统 Postgres 应用场景

fs9 对 parquet/csv/jsonl 都有特别支持，内嵌全文检索和 GIN（倒排索引），自动向量化和向量索引，还有 cron job 和 http call 能力。

## 什么是真的 Agent 友好？

- 无缝的 skill onboarding
- 永远不会因为没有 traffic 就 pause 掉你的 db
- 支持匿名数据库（连注册都不需要）
- 基于百亿级别 Agent 租户的设计（1/100 成本）
- 底层基于 TiDB X 的云原生引擎

## 应用案例

**mem9.ai**：给 OpenClaw 等 agent 用的持久化记忆基础设施。每只龙虾的记忆后端就是一个独立的数据库。未来龙虾数量可能是几十亿规模，db9 可以为每只提供免费的独立 postgres。

**mails.dev**：CLI/Skill first 的邮件服务。为每个 agent 注册一个邮箱，通过邮件服务在全网找到这只 agent。每个邮箱对应一个 db9 数据库实例。
