# Anthropic 最新报告：大模型并没有拉平一切，用户能力差距正在拉大

Anthropic 刚刚发布了 2026 年 3 月的《人类经济指数报告》（Economic Index report: Learning curves）。这份报告的数据样本采集节点非常微妙，即 2026 年 2 月上旬，刚好是 Claude Opus 4.5 发布三个月后，与 Opus 4.6 的面世完全重叠。如果我们只把这份报告当成一个简单的“大模型用户画像”看，那可能会错过当下 AI 生态中最重要的一条暗线：AI 并没有像很多人吹嘘的那样“拉平一切”，相反，它正在以前所未有的速度加剧用户之间的“阶层固化”。这不是危言耸听，数据即证明。网页端任务分散，高价值编码任务向 API 转移报告揭示的第一个极其割裂的现象是，Claude.ai（网页端）的用例正在迅速分散，而硬核生产力正在逃离。数据明确指出，2 月份网页端排名前 10 的常见任务占比，从去年 11 月的 24% 直接掉到了 19%。大量新涌入的用户在网页端干嘛？数据追踪到了海量关于体育比分、产品比较和家庭维护的“个人提问”。按平台（Claude.ai 与 1P API）划分的前十大任务使用份额随时间的变化。按平台和报告版本划分的、归属于十大最常见 O*NET 任务的对话占比。与之形成鲜明对比的是，以“写代码”为代表的高价值任务，正在持续从网页端的辅助工具，转变为第一方 API（1P API）流量中更自动化的工作流。Claude Code 的代理架构（Agentic architecture）将复杂的编码工作拆分成了无数个微小的 API 调用。网页端越来越像一个满足好奇心的通用搜索框，而真正的硬核业务流早已在底层 API 扎根。任务平均价值下滑，不同群体使用场景分化随着网页端用例的“下沉”，Claude 上所完成工作的平均经济价值（以美国相关职业的时薪衡量）出现了实质性的下降：任务估值从 49.3 美元降至 47.9 美元。不同版本与平台间平均任务价值的变化。 该图表使用 O*NET 框架，估算了在 Claude.ai 和第一方 API（1P API）上所执行任务的美元价值。任务价值是以从事该任务的工人所获得的平均时薪来估算的。这是一个标准的“技术采用曲线（Adoption curve）”在现实中的残酷投影：早期那批技术极客和专业人士，将 AI 牢牢绑定在“写代码”、“数据分析”等高杠杆、高时薪的特定高价值场景；而随着 AI 逐渐破圈，后来涌入的大众用户，只能将其用于范围更广、但经济附加值极低的日常琐事。高价值与低薪任务的折叠，清晰地划出了两类用户的生产力鸿沟。全球 AI 使用量的地区差异持续扩大如果说个体层面的场景折叠还不直观，宏观层面的数据则更加令人警醒：全球人工智能的采用差距不仅没有缩小，反而在一路狂飙。地理趋同。 该图展示了美国各州（左图）和各个国家/地区（右图）Anthropic 使用指数的洛伦兹曲线。报告显示，排名前 20 的国家，直接吃掉了全球 48% 的人均使用量（比上一期报告的 45% 还要高）。AI 的早期红利高度集中在高收入国家和知识工作者密集的区域。技术原本被寄予“消除不平等”的厚望，但在缺乏互补技能（complementary skills）和专业知识的地区，这道鸿沟正在被大模型无情地拉大。用户根据任务难度与成本，匹配不同的模型在具体的使用策略上，高阶玩家展现出了极其冷酷的工程化思维：他们绝不会用最贵的模型去解最简单的题。模型选择与职业领域。 该图表展示了付费 Claude.ai 用户在选择 Opus 级别模型时，根据任务所属的职业领域不同，其选择比例偏高或偏低的程度。在该样本中，Opus 占总使用量的 51%，而在计算机与数学领域的使用中，Opus 的占比为 55%（高出 4.4 个百分点）。报告揭示，在付费用户中，面对劳动力市场里高薪的“编码任务”，调用最强模型 Opus 的比例比平均水平高出 4 个百分点；但面对简单的“辅导类”任务，Opus 的出场率直接骤降 7 个百分点。到了程序化调用的 API 端，这种对任务复杂度的敏感响应更是翻倍：任务的时薪价值每增加 10 美元，切给 Opus 处理的比例就会猛增 2.8 个百分点。成熟的玩家早就在构建精细的模型路由矩阵，将模型能力与当前任务完美匹配，而新手还在为单一模型的偶尔幻觉而焦虑。资深用户减少单向指令，通过增加迭代提高成功率这是整份报告最刺眼的一组数据，也揭示了“学习曲线”的真正威力。更重要的是，它直接推翻了行业内一个巨大的迷思：用 AI 越熟练的高手，是不是越喜欢丢一个指令让大模型“全自动（Automation）”干完？Anthropic 的数据给出了否定的答案：注册使用 Claude 超过 6 个月的资深老玩家，他们使用单向“指令性（Directive）”委派任务的比例大幅下降。相反，他们更多地利用 Claude 进行任务迭代（Task iteration）和构建反馈循环。 他们不追求“撒手不管”，而是退一步，成为了深度介入的监工。经验与成功之间的关联。 该图展示了在日益严格的控制变量下，将我们的二元成功指标对“高资历（长使用时间）”指示变量进行回归分析的结果。回归系数以百分点为单位，误差线（须线）代表 95% 的置信区间。“Task FEs（任务固定效应）”表示 O*NET 任务和请求集群的固定效应。“全面控制变量”（Full controls）则进一步加入了模型、应用场景以及国家/地区的固定效应。这种“边做边学（Learning-by-doing）”带来了压倒性的胜率。高资历玩家在对话中的“成功率”硬生生比新手高出了 10%（即便加入最严苛的干扰变量控制，依然高出 4 个百分点）。在这个极其早期的红利阶段，有效使用 AI 需要极强的互补技能。那些更早入场、更高频试错的用户，已经养成了一套能精准榨取 AI 能力的习惯和策略。这五个变化，拼凑出了一个客观的技术现状：大模型时代的初期收益是不均衡的。正如报告中经济学家所指出的，一场“技能偏向型技术变革（Skill-biased technological change）”已经全面展开。老玩家通过不断迭代拉开的成功率差距，以及核心任务向 API 转移背后的工程化趋势，反映出了普通用户与资深用户之间正在形成的能力差距。看完这份报告，就会明白为什么经济学家开始担忧“技能偏向型技术变革”。AI 并没有成为人人平等的工具。相反，它放大了原有的技能差异。越早使用、越懂得如何匹配模型、越明白何时该深度参与迭代，在技术发展中建立的优势就越大。当部分用户还在把 Claude 当成高级搜索引擎时，高阶玩家已经在 API
