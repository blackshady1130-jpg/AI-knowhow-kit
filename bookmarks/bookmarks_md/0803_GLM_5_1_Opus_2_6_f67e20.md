# GLM-5.1 上线，与 Opus 仅差，2.6 分

智谱这次，不按常理出牌……没发技术报告，没放 benchmark 长图，直接宣布：“ GLM-5.1 即刻面向所有 Coding Plan 用户开放。一句话，就这么上线了。这操作，很有点当初 DeepSeek 的风格啊，不搞营销造势，直接开放使用，用体验来说话。012.6 分的差距先来看一级最核心的数据。GLM-5.1 编程评测对比：与 Opus 4.6 仅差 2.6 分智谱用 Claude Code 作为测试工具，跑了一套编程评测。结果是：• Claude Opus 4.6：47.9 分• GLM-5.1：45.3 分• GLM-5：35.4 分GLM-5.1 和 Opus 4.6 之间，只差了 2.6 分，达到了 Opus 性能的 94.6%。而跟上一代 GLM-5 相比，则从 35.4 跳到 45.3，一次更新提升了 28%，将近 10 分的飞跃。换个说法，GLM-5 还在及格线附近徘徊的时候，GLM-5.1 已经逼到了班里第一名的背后。在 SWE-bench Verified 上，GLM-5 系列拿到了 77.8%，和 Opus 4.6 的 80.8% 差距在 3 个百分点以内。在 Artificial Analysis 的智能指数榜单上，GLM-5 是首个达到 50 分的开源模型，排在 65 个模型中的第一梯队。在 LMArena（Chatbot Arena）的人类偏好评分中，GLM-5 拿到了 1451 分，也是开源模型里的第一。一个开源模型，在编程能力上追到了闭源之王 94.6% 的水平。02实测对决光看数据还不够，当然得看上手实测。博主 stevibe 做了一个 head-to-head 的对比测试：让 GLM-5 和 GLM-5.1 分别从一个基础 HTML 模板出发，构建一个交互式 Apple Watch 主屏幕，迭代 3 轮，逐步加入鱼眼物理效果、动量拖拽和应用图标。这个测试同时考验了布局计算、触摸交互、动画流畅度和多轮指令执行能力。第一轮：基础布局。两个模型都完成了蜂巢排列的圆形网格，差距还不大。第一轮：GLM-5 vs GLM-5.1 基础布局对比第二轮：加入鱼眼物理和动量拖拽。差距开始拉开了。GLM-5.1 的鱼眼缩放更自然，拖拽时的惯性和回弹也更流畅。而 GLM-5 的交互……看起来明显僵硬一些。第二轮：加入物理交互后的差距第三轮：替换成真实 App 图标。结果……GLM-5.1 成功放上了 20 多个风格统一的 emoji 图标，带标签、有层次感。GLM-5 这边虽然也放上了图标，但密度和精细度差了不少。第三轮：最终效果对比GLM-5.1 的优势在于长程任务和复杂物理模拟，这让它在迭代式的、细节丰富的 UI 构建上表现更好。YouTube 博主 AICodeKing 在实测之后的评价是：“ GLM-5.1 是目前最好的 agent 模型之一。我测试了它，指令遵循能力和长时间运行任务的表现，比上一代好太多了。AICodeKing 的多模型编程排行榜在他的实测排行中，GLM-5.1 + Kilo CLI 组合拿到了综合第二名（85.0 分），仅次于 Sonnet 4.6 + Kilo（87.9 分），排在 Opus 4.6 + Claude Code（83.6 分）前面。03我也试了一把看别人的测试总觉得，不够真实，差点意思，所以我自己也上手试了试。我之前做了个小项目叫 plaw.social，一个让 AI Agent 替我社交的平台，招聘、求职、找合伙人、交友，Agent 先聊，合适了再见面。过年期间上线之后我就一直没怎么管了，没想到回头一看，居然还有十多个人在用。plaw.social 改版前的样子（呃、、有点乱码问题啊刚发现）页面嘛……功能是有了，但说实话，我总感觉不够酷，有点糙。我一直想优化一下前端，让它更好看、更酷炫一点，但一直没下手。这次看到 GLM-5.1 上线，我就想：要不让它帮我搞搞呢？正好这两天一个叫 pretext 的开源项目火了，Midjourney 工程师 Cheng Lou 做的，一个 15KB 帮我升级后的效果背景加上了绿色粒子光晕效果，还有个光球在用 Lissajous 曲线做二维运动，上下左右都会游走，还会跟随鼠标有相关的动效。整个页面，瞬间就有了那种赛博朋克的氛围。而整个过程，我基本就是发了几句话，然后就忙其他去了，然后，GLM-5.1 就把活干完了。对于这种级别的综合型难度的优化任务，GLM-5.1 用起来确实信手拈来，也算是轻松得很了。04海外「炸锅」了这次 GLM-5.1 的发布，在海外社区引起的反响似乎比在国内要大得多。科技博主 X Freeze 写了一段话，语气显得非常地不客气：“ 中国开源模型将会把 Anthropic 和 OpenAI 远远甩在身后。开源和闭源之间的编程差距，实际上已经消失了。GLM-5.1 的编程性能几乎与 Claude Opus 不相上下，但价格低约 10 倍。X Freeze 对 GLM-5.1 的评价他还观察到一个现象：大量开发者已经在本地运行 GLM 模型来替代 Claude，或者付费使用更便宜的 API 推理，只是这些人还没有在 X 上公开讨论。BridgeBench 的测试也显示，在跟 Qwen 3.6 Plus Preview 的对比中，GLM 5.1 和 Claude Opus 4.6 在 UI 创意和完成度上仍然有明显优势。GLM-5.1 vs Opus 4.6 性价比对比从定价来看，GLM-5.1 的 API 价格大约是输入 $1/百万 token，输出 $3.2/百万 token。而 Opus 4.6 的价格是输入 $15，输出 $75。差了大概，15 到 23 倍。而 Coding Plan，最低则只要 $3/月（促销价），$10/月起步。每次都是上线即售罄，也算是罕见了。05速度换智力当然……GLM-5.1 目前还存在些短板。速度 vs 智力的取舍BridgeBench 在测评后的结论是：“ GLM 5.1 是我们测过的最慢的前沿模型。44.3 tokens/秒。GPT 5.4 速度的一半。Grok 4.20 的六分之一。Z.ai 用所有的速度换了智力。BridgeBench 速度测评stevibe 等博主在测试中也有类似的反馈。这其实是个有争议的取舍。GouthamRajesh4 表示这并不那么重要：“ 它便宜，而且比快模型好得多。开 10 个 agent，去睡觉，醒来产品就做好了。睡着就能赚钱。不过，速度瓶颈可能不在模型本身，而在 Z.ai 的推理基础设施和卡的数量上、以及用户的热情上，相信速度很快会有改善。模型本身的能力提升了，但速度问题并不难解决，瓶颈在基础设施，并非模型本身。06长程任务赛道而为什么智谱这次，会把重心放在编程和 agent 能力上呢？从整个行业的方向上看，AI coding 正在经历一条清晰的能力跃迁路径。最早是 AI coding，本质上是程序员的效率工具，模型帮你补全代码、调用工具，但它主要还是个辅助者。然后是 vibe coding，coding 从专业行为变成了一种更大众的表达方式，人们不需要理解每一行代码，但可以把想法快速变成产品原型。再往前是 agentic，AI 能自主理解需求、制定计划、编写代码、测试并迭代修复。而下一个阶段，就是 long horizon（长程任务）。AI Coding 能力跃迁路径METR 的研究也印证了这个趋势。他们发现，AI 能以 50% 成功率完成的任务复杂度（以人类专家耗时衡量）正在指数级增长，2024-2025 年加速到了大约每 4 个月翻一倍。目前 Claude Opus 4.6 能稳定完成的任务，大约相当于人类专家工作 12 小时的工作量。如果趋势持续，2027 年 agent 可能就能搞定需要一个月才能完成的任务了。GLM-5.1 瞄准的，也正是这个方向。它在长时间跨度、多步骤依赖、多工具协同、持续执行和目标保持等能力上做了系统性优化。用智谱自己的话说，目标是让模型像资深工程师一样交付完整工作。实际效果上，从 AICodeKing 的测试来看，GLM-5.1 在多轮迭代任务中的完成度确实有了明显提升，尤其是在长链路的后半程，上一代容易断链的问题得到了改善。07赶到后背说了这么多优点，当然也得客观看看目前还差在哪里。BridgeBench 的幻觉测试显示，GLM-5 的幻觉率是 38.6%，GLM-5.1 降到了 34.1%。一次更新降了 4.5 个百分点，改进幅度也是相当不小。但跟 Opus 4.6 比，这个数字还有优化空间。BridgeBench 幻觉率对比而速度的问题前面分析过了，更多是基础设施层面的事，模型本身的推理能力并不受影响。随着 Z.ai 扩容和更多推理服务商接入，这个问题应该会逐步缓解。另外，GLM-5.1 目前还没有视觉理解能力（不支持图像输入）。不过按照 GLM 系列过往的节奏，视觉版本应该很快就会跟上，毕竟 GLM-5V 也刚已经放出来了。08何时开源目前 GLM-5.1 只能通过 Coding Plan 使用，还没有正式开源。但海外社区已经在狂催了，「什么时候开源」是被问得最多的问题之一。Z.ai 全球负责人李子轩在 X 上确认 GLM-5.1 会开源，但目前仍没给出时间线。GLM-5 系列用的是 MIT License，也就是说，一旦开源，商用门槛基本为零。考虑到 GLM-5 的架构是 744B 参数的 MoE 模型（每次激活 40B），用 256 个专家中的 8 个，而且是在 10 万块华为昇腾 910B 上训练的，没用一块 NVIDIA GPU……所以，这个开源的意义，可能比模型本身的性能还要大。09怎么用如果你已经是 Coding Plan 用户，切换到 GLM-5.1 并不复杂。在 Claude Code 里，打开 ~/.claude/settings.json，把模型名改成 glm-5.1 就行：●●●{  "env": {
