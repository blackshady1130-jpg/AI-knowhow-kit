# 检索质量分与权重设计说明

版本：v0.1  
更新日期：2026-03-15

## 1) 为什么需要质量分层

本知识库包含 800+ 条资料，质量参差不齐。如果 Agent 不加区分地召回，低质量条目会稀释高质量证据，导致回答可信度下降。

质量分层的目标：**让 Agent 在有限的上下文窗口中，优先加载信息密度最高的材料。**

## 2) 两个知识库采用不同的质量体系

| 维度 | AI 行业扫描（`notes/`） | Bookmarks（`bookmarks/`） |
|---|---|---|
| 定位 | 高信号精选，每条都有人工判读 | 广覆盖收藏，质量波动大 |
| 质量信号 | `type`（文章类型）+ `why`（人工判读理由） | `quality_tier`（A/B/C）+ `retrieval_weight`（数值权重） |
| 条目数量 | ~187 条 | ~684 条 |
| 为什么不同 | 每条都经过人工筛选和判读才收录，本身就是"已过滤"的，不需要再分级 | 收藏来源广泛，包含高质量深度文章和低信息量页面，必须分级 |

### AI 行业扫描的质量信号

**`type` 字段**（文章类型）：

| 类型 | 含义 | 数量 | 信号强度 |
|---|---|---|---|
| `solid paper` | 有实验/数据支撑的论文或技术报告 | 39 | 最高——结论有可复现依据 |
| `观点/文章` | 深度分析、观点长文、行业评论 | 82 | 高——有框架和判断，但需注意作者立场 |
| `行业动态/评价` | 新闻、产品发布、市场变化 | 62 | 中——信息新鲜但时效性强，过期快 |

**`why` 字段**（人工判读理由）：

这是 AI 行业扫描中最有价值的字段。它记录的不是文章摘要，而是**你为什么认为这篇值得收录、核心启发是什么、与已有认知的关系**。Agent 在筛选候选时，`why` 的信息密度远高于 `title`。

### Bookmarks 的三级质量体系

**`quality` 字段**（人工评级，4 档）：

| 等级 | 含义 |
|---|---|
| `高质量` | 原创深度分析、有数据/实验支撑、观点密度高、值得反复引用 |
| `良好` | 信息整合扎实、有清晰框架、内容完整、值得回查 |
| `中等` | 信息有用但密度一般，或覆盖面窄 |
| `较短` | 内容过少（通常 < 1000 字），仅作补充参考 |

**`quality_tier` 字段**（机器可读分级，3 档）：

将 4 档 `quality` 映射为 3 个检索层级：

| quality_tier | 对应 quality | 检索行为 |
|---|---|---|
| `A` | 高质量 | 默认优先召回，直接作为证据 |
| `B` | 良好 | 默认召回，与 A 一起参与筛选 |
| `C` | 中等 + 较短 | 降权处理，仅在 A/B 不足时补充 |

**`retrieval_weight` 字段**（数值权重）：

| quality_tier | retrieval_weight | 设计意图 |
|---|---|---|
| A | `1.0` | 满权重，无衰减 |
| B | `0.85` | 轻微衰减，仍属于主检索池 |
| C | `0.6` | 明显衰减，进入候选池但不默认优先 |

权重的使用方式：**不是数学公式中的乘数**，而是给 Agent 的语义信号——当多个候选相关性接近时，优先选择 `retrieval_weight` 更高的条目。Agent 不需要做加权计算，只需要理解"A > B > C"的优先级。

## 3) Agent 的检索决策流程

```
用户问题
    │
    ▼
Step 1: 关键词匹配 ─── 在 jsonl 索引中召回候选（5-10 条/库）
    │
    ▼
Step 2: 质量筛选
    ├── AI行业扫描：按 type 和 why 判断相关性与可用性
    └── Bookmarks：按 quality_tier 筛选（默认 A/B 优先）
    │
    ▼
Step 3: 相关性排序 ─── 综合 title/summary/keywords 与问题的匹配度
    │
    ▼
Step 4: 定向取证 ─── 仅读取最终 3-8 条的 md 原文
    │
    ▼
Step 5: 结构化输出 ─── 结论 → 证据 → 边界 → 下一步
```

## 4) 质量评级的操作规范

### 谁来评级

- `quality` / `quality_tier`：由你（知识库维护者）在收录时人工判定
- 不建议让 LLM 自动评级——模型倾向于给"看起来专业"的内容高分，但实际信息密度可能不高

### 评级标准（决策树）

```
文章是否有原创数据/实验/深度分析？
  ├── 是 → 高质量（A）
  └── 否 → 文章是否有清晰框架、完整信息整合？
              ├── 是 → 良好（B）
              └── 否 → 内容是否超过 1000 字且有可用信息？
                          ├── 是 → 中等（C）
                          └── 否 → 较短（C）
```

### 边界情况

- 同一篇文章有深度段落但整体较散：评为 `良好`（B），不要因为局部亮点拔高到 A
- 知名作者/机构但内容空泛：仍按内容实质评级，不因来源加分
- 翻译/转载文章：按翻译后的信息完整度评级，不是按原文质量

## 5) 常见问题

**Q: 为什么 AI 行业扫描不用 quality_tier？**

因为 AI 行业扫描的每条记录都经过你的人工筛选才收录——本身就是一次质量过滤。再做质量分级是冗余的。它的价值分层体现在 `type`（论文 > 观点 > 动态）和 `why`（判读深度）。

**Q: retrieval_weight 是否用于数学排序？**

不是。它是给 Agent 的优先级信号，不参与公式计算。Agent 在多个候选相关性相近时，参考这个字段做取舍。

**Q: 如果所有高质量条目都不相关，可以用 C 级条目吗？**

可以。C 级是"降权"不是"排除"。当 A/B 级无命中时，C 级条目正常参与召回，但输出时应标注"证据覆盖有限"。

**Q: 新增条目时，拿不准该评 A 还是 B 怎么办？**

默认评 B。宁可低估后续升级，不要高估导致 Agent 过度信任低密度内容。
