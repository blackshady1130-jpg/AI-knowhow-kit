# MAINTENANCE

## 月度更新节奏（建议每 4 周）

1. 增量收集：新增博客、分析表、对话导出。
2. 标注变化：把“新偏好/新目标/新术语”单独记录。
3. 去噪删除：删掉一次性情绪、临时任务上下文、过时观点。
4. 更新四个核心文件：
   - `PROFILE_CORE.md`：只更新稳定信息
   - `WORKING_PREFERENCES.md`：只更新重复出现的偏好
   - `DOMAIN_KNOWHOW.md`：补框架，不堆链接
   - `STYLE_GUIDE.md`：补“常见改写前后对照”
5. 回归测试：选 3 个高频任务做 A/B 对比（有无该资料包）。

## v0.2 维护要求

1. 先更新 `docs/sources/` 审计文件，再更新 core 文件。
2. 若画像条目无证据复现，降级到“中置信”或移除。
3. 如果出现身份冲突，新增到 `PROFILE_CORE.md` 的“记忆卫生规则”。

## A/B 评估指标（建议）

- 首稿可用率（是否可直接进入文档）
- 平均改写轮次
- 任务完成时间
- 幻觉/不符偏好次数

## 版本管理建议

- 文件头保留 `版本 + 日期 + 变更摘要`。
- 每次改动不超过 20% 内容，避免大幅漂移。
- 对争议性判断加上“适用范围”和“失效条件”。

## 行业扫描知识库月更动作（新增）

建议每月固定执行以下 4 步：

1. 增量导入：补充当月新增条目（至少含 `id/date/type/title/why/url/md_path/keywords`）。  
2. 关键词重算：更新关键词与主题标签，清理低信息量或重复标签。  
3. 路径校验：确保主索引 `md_path` 全为相对路径（`AI行业扫描_md/...`）且文件可达。  
4. 抽样回归：用 5 类问句（公司/模型/评测/Agent/SaaS）验证回答可追溯（含 `id/title/url`）。  

## 行业扫描维护验收

- 主索引无绝对路径、无失效 `md_path`。  
- 检索流程遵循“先召回、再判读、后取证”。  
- 结构化输出保持 `结论 -> 证据 -> 边界 -> 下一步`。  

## bookmarks 知识库月更动作（新增）

建议每月固定执行以下 4 步：

1. 增量导入：同步最新 `xlsx` 与 `bookmarks_md`，补充新增条目。  
2. 关键词更新：重跑或补充 `LLM关键词`，清理低区分度词。  
3. 索引重建：刷新 `bookmarks_keywords.jsonl`，并校验 `md_path` 相对路径可达。  
4. 抽样回归：用 5 类问题（公司/模型/评测/Agent/SaaS）验证可追溯输出。  

## bookmarks 维护验收

- `bookmarks_keywords.jsonl` 与 `bookmarks_md` 条目数量一致。  
- 主索引 `md_path` 全部为相对路径 `bookmarks_md/...`。  
- 入口路由能命中 `BOOKMARKS_RETRIEVAL.md` 与双库协同策略。
